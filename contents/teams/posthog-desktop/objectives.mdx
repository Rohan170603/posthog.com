## Q3 2026 goals

<details>
<summary>Objective 1: Get PostHog Desktop into general availability</summary>

- **What we'll ship:** Multiplayer agent sessions, channels, thread, generative UI and artifacts as one unified product. Some features will still carry a beta label.

</details>

<details>
<summary>Objective 2: Move to usage-based pricing</summary>

- **What we'll ship:** Usage-based billing on token and compute costs, plus usage visibility and cost controls with sensible defaults so users can understand and manage what they spend.

</details>

<details>
<summary>Objective 3: New features that differentiate us</summary>

- **What we'll ship:**
  - Loops and workflows, for coding and non-coding work
  - Product harness, with data driven plan mode so plans are grounded in real product data from the start
  - More models, including open-source models for cheaper execution

</details>

<details>
<summary>Objective 4: Reliability & performance</summary>

- **What we'll ship:**
  - Real end-to-end integration tests so changes stop causing regressions
  - First-principles rebuilds where we've been carrying inherited constraints and band-aid patches
  - Performance work: faster task creation and switching, app and agent boot times, and lower CPU/RAM usage
  - Rock solid cloud agents
  - A real process for the bug backlog and PR reviews

</details>

### Stretch objectives

- Web version of PostHog Desktop
- Memory: ambient product awareness in every agent (Raquel, working across teams on shared context)
- Apps as artifacts

### Bets we're making

- Most agent execution will happen in the cloud, for coding and non-coding work alike
- Open-source models will keep getting better and drive execution costs down
- Engineers won't primarily collaborate via Git anymore, collaboration happens _while_ the work is being done
